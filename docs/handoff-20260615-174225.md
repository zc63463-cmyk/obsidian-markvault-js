### 架构要点

**颜色解析链**：
```
Annotation.color → resolveColor()
  ├── '#xxxxxx' (hex) → 直接返回
  ├── 'yellow' (preset ID) → PRESET_COLOR_MAP → '#FACC15'
  └── undefined → getKindColor(kind) → KIND_COLORS[kind] || '#6b7280'
```

**节点去重**：
- 主循环：`if (!nodeMap.has(ann.uuid))` — 第一次遍历到的标注不会重复添加
- 目标节点创建：`if (!nodeMap.has(rel.targetUuid))` — 关系目标不会重复添加
- 已有节点更新：`Object.assign(nodeMap.get(ann.uuid)!, node)` — 保留位置，更新属性

**主题适配**：
- `detectTheme()` 检测 `body.classList.contains('theme-dark')` / `'theme-light'`
- 6 个颜色槽位：highlightStroke / normalStroke / labelBg / labelTextBright / labelTextDim / iconText
- 每次 `refresh()` 自动检测主题变化

**邻接表**：
- `refresh()` 时 `rebuildAdjacencyMap(graphData.links)` 构建 `Map<nodeId, {nodes, links}>`
- `handleNodeHover()` O(1) 查询 vs 旧版 O(n) 遍历全量 links

### Phase 3: W3C Web Annotation 互操作导出

**目标**：将标注数据导出为 [W3C Web Annotation Data Model](https://www.w3.org/TR/annotation-model/) 格式，可被 Hypothesis 等工具消费。

**核心任务**：
1. **W3C Annotation JSON-LD 序列化器**
   - 每条 Annotation → W3C Annotation 对象（`@context`, `id`, `type`, `body`, `target`）
   - `target` 的 `selector` 需支持：TextQuoteSelector（contextBefore/contextAfter）、TextPositionSelector（startOffset/endOffset）、RangeSelector（region）
   - `body` 需支持：TextualBody（note/comment）、Tag（tags）、motivation 映射
   - Relation → `dc:relation` 或自定义扩展

2. **导出 API**
   - `exportToW3C(filter?)` → W3C AnnotationCollection（含 `@context` + `items[]`）
   - 支持按文件/关系类型/motivation 过滤
   - Container 格式：AnnotationPage（分页支持）

3. **导入 API（可选）**
   - 从 W3C Annotation Collection 导入回 MarkVault
   - UUID 映射策略（如何将外部 id 映射为内部 uuid）

4. **参考项目学习**（用户要求每个 phase 都从 GitHub 学习）
   - [hypothesis/client](https://github.com/hypothesis/client) — W3C 注释标准的工业级实现
   - [annotorious/annotorious](https://github.com/annotorious/annotorious) — W3C 注释工具
   - [w3c/web-annotation](https://github.com/w3c/web-annotation) — 官方测试套件和规范

5. **测试**
   - W3C JSON Schema 验证（导出结果必须符合 W3C 规范）
   - 往返测试：MarkVault → W3C → MarkVault 数据无损
   - Relation 序列化/反序列化

### 后续 Phase

6. **Phase 5: LLM-ready 导出系统**
   - ExportEngine 独立模块 + 4 种格式（JSON/Markdown/CSV/LLM Prompt）
   - Prompt 模板系统（掌握度评估/闪卡生成/隐含关联发现/理解纠偏）
   - 参考 `.workbuddy/memory/MEMORY.md` Phase 5 规划

7. **索引优化**
   - `queryAnnotations` 的 `hasRelations`/`needsCorrection`/`motivation` 未走索引捷径
   - 可为这些维度添加专用倒排索引
