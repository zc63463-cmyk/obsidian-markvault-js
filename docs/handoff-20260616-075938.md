## 一、本项目最后完成的工作 (v5.11)

### 做了什么

用户指出关系类型色块有两大问题：
1. 多个类型色调相似难区分（泛化/特化/部分同紫色，证明/补充同绿色）
2. 芯片无序排列，无语义分组逻辑

**解决方案 — 6 维度分层着色 + 分组渲染：**

| 维度 | 包含类型 | 色系 | 新颜色 |
|------|---------|------|--------|
| Taxonomic 分类 | 泛化/特化/部分 | Indigo→Violet→DeepPurple | #4F46E5 / #7C3AED / #6D28D9 |
| Argumentative 论证 | 证明/反驳/对比 | Green(+), Red(-), Amber(±) | #16A34A / #DC2626 / #CA8A04 |
| Expositive 阐释 | 详述/举例/图示 | Dark Amber→Gold→Orange | #A16207 / #EAB308 / #EA580C |
| Referential 引用 | 引用/应用 | Cyan→Blue | #0891B2 / #2563EB |
| Dynamic 动态 | 使能/导致/先于 | Teal→Rose→Sky | #0D9488 / #E11D48 / #0284C7 |
| Structural 结构 | 关联/补充 | WarmGray / Emerald | #78716C / #10B981 |
| Passive 被动 | 全部14个反向类型 | Cool Gray | #9CA3AF |

## 二、当前项目已知问题

### 🔴 P0 — Obsidian 内部报错（非本插件导致）

```
RangeError: Field is not present in this state
    at e.field (app.js:1:298157)         ← CodeMirror State.field()
    at t.saveHistory (app.js:1:2412322)   ← Obsidian 编辑器历史保存
    at t.beforeUnload (app.js:1:2873850)  ← 切换/关闭文件时
```

**分析:**
- 所有栈帧都在 Obsidian 内部 `app.js:1`，不是本插件的 `main.js`
- 本插件唯一调用 `editor.dispatch()` 的地方在 `src/core/highlight-applier.ts:68`（旧代码，已 try/catch）
- 这是 Obsidian/CodeMirror 已知类型 bug：多插件 CM StateField 交互时，一个扩展的 field 在编辑器重建后丢失引用

**用户反馈的触发场景:** 未明确 — 可能是切换标签页时偶发

**建议排查路径:**
1. 二分法禁用其他社区插件，定位是哪两个插件冲突
2. 检查 Obsidian 版本是否为最新
3. 打开 DevTools (`Ctrl+Shift+I`) 查看完整上下文

### 关系类型系统（Schema-First）

- **30 条 RelationTypeConfig 注册项**: 16 active (用户可选) + 14 passive (系统自动维护)
- 配置存储在 `PluginSettings.customRelationTypes`，运行时动态构建 `RelationSchema` 实例
- 反向关系自动创建（如创建 `A → applies → B` 时自动创建 `B → isAppliedBy → A`）
- 对称关系 (`isSymmetric: true`): contrasts, supplements, associates — reverseId 等于自身
- 时态字段: `invalidAt` (软删除), `source` (manual/template/inferred/imported)
- 被动关系统一用灰色系，与 active 视觉区分

### 图谱功能完整堆栈

```
DEFAULT_FILTER (7 个筛选维度)
  │
  ├─ buildGraphData(annotations, filter, schema)
  │   ├─ showInvalidated 过滤
  │   ├─ bfsReachable 邻居深度过滤
  │   ├─ relationTypes / filePaths / annotationKinds 过滤
  │   └─ per-link _distance = 35 + Math.min((srcLen+tgtLen)*6, 160)
  │
  └─ RelationGraphView
      ├─ ForceGraph Canvas (d3-force)
      │   ├─ per-node charge: -80 - degree*15
      │   ├─ forceCollide: 碰撞检测含标签区域
      │   └─ 三档缩放标签: 10/sqrt(globalScale) ∈ [6,14]
      ├─ Toolbar (4 行)
      │   ├─ Row1: 标题 + 搜索 + 刷新/适配/重置
      │   ├─ Row2: Relation chips (语义分组 + 分隔线)
      │   ├─ Row3: Kind chips (inline/block/span/region)
      │   └─ Row4: Isolated/Invalid toggles + Depth slider + File select + Stats
      └─ 交互: hover 详情 / click 定位 / drag 固定 / zoom/pan
```
