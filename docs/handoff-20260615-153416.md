### 关键技术决策

1. **增量索引 vs 全量重建**：关系级操作（addRelation/removeRelation/invalidateRelation/restoreRelation）必须用增量索引，`_removeFromIndex` + `_addToIndex` 仅限标注级 CRUD
2. **软删除模型**：`invalidAt` 标记失效而非物理删除，保留历史可回溯（与 filter-engine 的 `!r.invalidAt` 过滤一致）
3. **双向自动维护**：用户只需创建 A→B，系统自动创建 B→A（reverseType），用户不需要知道反向类型
4. **三元组去重**：`targetUuid + type + !invalidAt` 判定重复

### 架构要点：`_removeFromIndex` / `_addToIndex` 的安全边界

| 调用场景 | 安全性 | 说明 |
|----------|--------|------|
| addAnnotation / deleteAnnotation / updateAnnotation | ✅ 安全 | 标注级全量操作 |
| updateFlags / addGroupTo / removeGroupFrom | ✅ 安全 | 标注级操作 |
| rebuildIndex | ✅ 安全 | 全量扫描 |
| addRelation / removeRelation / invalidateRelation / restoreRelation | ✅ 安全 | 已全部改为增量索引 |
