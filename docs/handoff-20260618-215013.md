## 1. 项目概述

在 MarkVault-JS (Obsidian 批注插件 v5.0.0) 内开发 MindFlow 思维导图子模块。已跨越 Phase 1 (MVP)、Phase 1.5 (交互增强)、Phase 2 (MD Inline 渲染)，当前 57 个单元测试全绿，但在真实 Obsidian 环境中有渲染 Bug。

## 4. 🚨 已知 Bug（最高优先级）

### Bug 1: 节点内容显示为纯文本（Markdown 未渲染）

**现象**: 节点显示原始 Markdown 文本，如 `**行列式定义**：$\det(A) = ...$`，而不是渲染后的粗体+公式。

**根因分析**: 
`MarkdownRenderer.render()` 在 Obsidian 内部调用全局 `MathJax` 对象渲染 `$...$` 公式。Obsidian 的 MathJax 是懒加载的——只在用户首次打开阅读模式下含公式的 .md 文件时才加载。MindFlow 视图打开时 MathJax 可能未加载 → `render()` 内部抛 `ReferenceError: MathJax is not defined` → `renderNodeContent` 的 `catch` 块走 fallback：

```typescript
// src/mindflow/render/node-renderer.ts:160
} catch (err) {
  contentEl.textContent = node.text; // ← 纯文本回退！
}
```

**已尝试的修复**: 在 `layoutAndRender()` 的 Step 2 前添加了 `loadMathJax()`（Obsidian API）：

```typescript
// src/mindflow/view/mindflow-view.ts:384
try {
  await loadMathJax();
} catch (err) { /* silent */ }
```

但可能仍然无效。**可能原因**:
1. `loadMathJax()` 在 try/catch 中被吞掉，实际加载失败但继续执行
2. `loadMathJax()` 返回的 Promise resolve 时 MathJax 全局变量尚未就绪
3. Obsidian 版本差异导致 API 行为不同

**修复方向**:
- 移除 `loadMathJax()` 的 try/catch，让错误抛出
- 如果 `loadMathJax()` 后 MathJax 仍不可用，用 `renderMath()` + `finishRenderMath()` 替代方案
- 或延迟渲染直到 MathJax 全局变量存在（polling），带超时回退

### Bug 2: Free 节点 parentId 基于行号脆弱

**现象**: 修改 frontmatter 后 Free 节点 parentId 失效 → "7 free node(s) lost their parent"

**根因**: MD-Seed 节点 ID 是 `md-{0-based-lineNumber}`。添加/删除 Free 节点会改变 frontmatter 行数 → 所有 MD-Seed 的 lineNumber 偏移 → parentId 断裂。

**修复方向**:
- 写入 frontmatter 时自动修正 parentId（比较新旧 MD-Seed 映射）
- 或使用 heading-text-hash 替代行号作为 MD-Seed ID
- 短期方案：提供 "Fix Free Nodes" 命令扫描并修正所有 orphan

### Bug 3: 节点宽度自适应不完整

**现象**: 长公式/代码块被截断或溢出

**当前状态**: 已增大 `NODE_WIDTH` 160→240px，移除了 `white-space:nowrap`。但 `renderedWidth` 缓存值在首次渲染后生效时需要重布局才能更新 DOM 宽度。

## 5. 架构决策记录

| 决策 | 理由 |
|------|------|
| **纯 DOM 方案** (非 SVG foreignObject) | contentEditable 零摩擦；Obsidian MarkdownRenderer 天然支持；CSS 动画可控 |
| **MarkdownRenderer.render()** (非 katex) | 零新增依赖；复用 Obsidian MathJax 配置；一次性处理全部 MD 语法 |
| **两遍布局** (估算→异步渲染→重布局) | 解决异步渲染的高度不确定性；骨架即时可见 |
| **RenderCache** (nodeId+text 哈希) | 避免已渲染节点重复调用 MarkdownRenderer |
| **parentId 行号方案** | ❌ 已证明脆弱，待替换 |
| **loadMathJax() 预加载** | 🟡 尚未验证有效 |
