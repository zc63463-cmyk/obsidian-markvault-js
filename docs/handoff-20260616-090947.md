## 3. 架构概览

### 3.1 数据流

```
Markdown 文件 → annotation-parser.ts → AnnotationStore (12 indexes)
                                           ↓
                                    annotation-repo.ts (Repository)
                                           ↓
                    ┌──────────────────────┼──────────────────────┐
                    ↓                      ↓                      ↓
           annotation-modal.ts    RelationGraphView.ts    search-engine.ts
           (标注编辑 UI)          (力导向图谱)            (BM25 搜索)
                    ↓                      ↓
           relation-picker-modal.ts   graph-data-builder.ts
           (关系选择器)               (图谱数据转换)
```

### 3.2 倒排索引 (12 个)

```
_byUuid → _byFile → _byKind → _byType → _byColor → _byTag 
→ _byMotivation → _byGroup → _byMastery → _byReviewPriority 
→ _byRelationOut → _byRelationIn
```

所有查询 O(1)。BFS 邻接表每次 refresh() 时 O(n) 重建。

### 3.3 关系系统

```
30 条配置 → RelationSchema → 运行时查询
  ├─ 16 active (用户可选) + 14 passive (系统维护)
  ├─ 双向自动维护：addRelation/create → 自动创建 reverse
  ├─ 软删除：invalidateRelation → invalidAt 时间戳
  ├─ 恢复：restoreRelation → 双向清除 invalidAt
  └─ 级联：类型删除 → invalidateRelationsByType
```

### 3.4 语义分组 (SEMANTIC_GROUPS)

| 组 | 维度 | 颜色 | 类型 |
|------|------|------|------|
| Taxonomic | 分类学 | Indigo-Violet-DeepPurple | generalizes, specializes, part-of |
| Argumentative | 论证 | Green-Red-Amber | proves, refutes, contrasts |
| Expositive | 阐释 | Amber-Orange | elaborates, exemplifies, illustrates |
| Referential | 引用 | Cyan-Blue | references, applies |

| Passive | 被动 | Gray | isAppliedBy, isReferencedBy, ... (11 types) |

## 7. 重要注意事项

### 7.1 force-graph 运行时行为
- `graphData()` 注入数据后，link 的 `source`/`target` 从 `string` 被替换为 `GraphNode` 对象引用
- 任何 `link.source as string` 在运行时都会得到 `[object Object]`
- **必须**用 `typeof link.source === 'string' ? link.source : link.source.id` 模式

### 7.2 CM6 竞态
- 插件注册了 `regionCacheUpdatedEffect` (StateEffect) + 2 个 ViewPlugin
- 在 `onunload()` 首行必须调用 `setActiveEditorView(null)` 切断引用
- `requestRegionLayerRedraw()` 需要 `state?.field` 检查 + try-catch 兜底

### 7.3 关系双向维护
- 全部关系操作（add/remove/invalidate/restore）都自动维护双向关系
- 反向关系通过 `RelationSchema.getReverse(type)` 查找
- 不需要手动创建反向关系

### 7.4 格式升级遗留
- Block annotation 测试中 1 个 fail 是 `updateBlockAnchor` 输出新格式（含 alias 段）但测试期望旧格式
- 不影响生产环境，属于格式升级的已知问题
