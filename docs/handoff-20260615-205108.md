### 架构要点

**W3C 导入 API 设计**：
```
importFromW3CString(store, jsonString, options?)
  └→ importFromW3C(store, parsedCollection, options?)
       ├→ 逐条 deserializeAnnotation(w3cAnn)
       ├→ UUID 冲突策略: 'skip' | 'regenerate' | 'overwrite'
       ├→ filePath 映射: options.filePathMap
       └→ Relations 批量重建: 第一轮导入标注 → 第二轮重建关联
```

**关联详情面板交互**：
```
🔗 N (徽章, 可点击)
  └→ toggle 展开/折叠 relPanel
       ├─ Outgoing 区: [证明] → "数据库范式" 📄概论  ← 可点击跳转
       └─ Incoming 区: [被证明] ← "第三范式"          ← 可点击跳转
```

## 项目关键架构速查

### 倒排索引体系（12 个索引）

| 索引 | Key | 用途 |
|------|-----|------|
| `_byUuid` | uuid → Annotation | O(1) 主键查找 |
| `_byFile` | filePath → Set\<uuid\> | 文件级批处理 |
| `_byKind` | kind → Set\<uuid\> | 类型过滤 |
| `_byType` | type → Set\<uuid\> | 高亮/批注分类 |
| `_byColor` | colorId → Set\<uuid\> | 颜色过滤 |
| `_byTag` | tag → Set\<uuid\> | 标签搜索 |
| `_byMotivation` | motivation → Set\<uuid\> | 语义过滤 |
| `_byGroup` | group → Set\<uuid\> | 分组过滤 |
| `_byMastery` | mastery → Set\<uuid\> | 掌握度过滤 |
| `_byReviewPriority` | priority → Set\<uuid\> | 复习优先级过滤 |
| `_byRelationOut` | sourceUuid → Set\<relationTarget\> | 出边查询 |
| `_byRelationIn` | targetUuid → Set\<relationSource\> | 入边查询 |

### 关系系统（Schema-First）

```
RelationTypeConfig → RelationSchema
  ├─ 13 内置类型 (8 主动 + 4 被动 + 1 对称 associates)
  ├─ getReverse(type) → 反向类型
  ├─ getLabel(type) → 显示标签
  └─ getActiveTypes() → 可选类型列表

双向自动维护: A→B 自动创建 B→A (反向类型)
对称类型: associates 自反 (reverse = self)
软删除: invalidAt 标记 + restoreRelation 恢复
级联: deleteAnnotation → _cascadeDeleteRelations 清理伙伴反向
```

### 持久化架构

```
分片 JSON: annotations/{hash}.json → 每个笔记文件一个分片
索引文件: annotations/_index.json → filePath→shard 映射
元数据: annotations/_meta.json → 全局统计 + schema 版本
Debounce: 2 秒脏文件写入 + flushAll() 强制同步
Checksum: 写入时计算 SHA-256 + .bak 备份链
```

### 命令注册入口

```
src/ui/editor/context-menu.ts → registerCommands(plugin)
  ├─ 标注创建/编辑命令
  ├─ 标注模板命令 (per-template)
  ├─ W3C 全部导出 / 当前文件导出 / 文件导入 (v5.0 新增)
  └─ ...
```
