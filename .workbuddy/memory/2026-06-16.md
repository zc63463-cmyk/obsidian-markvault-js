# 2026-06-16

## Mermaid PanZoom V4 重写完成

基于三个参考项目 (mermaid-view-enhancer, obsidian-mermaid-fullscreen, Mermaid-Zoom-Viewer) 的深度研究，彻底重写了 `_openMermaidPreview()` 方法。

### V3 致命 Bug (已修复)
1. **事件监听器丢失**: mousemove/mouseup 在创建时注册到 document，首次 mouseUp 时移除后不再注册 → 第二次拖拽就失效 (崩溃根因)
2. **CSS transition 始终生效**: `.markvault-mermaid-canvas` 的 `transition: transform 0.15s ease-out` 在拖拽/滚轮时仍启用 → rubber-band 延迟 (卡顿根因)
3. **无 rAF 节流**: wheel/mousemove 直接操作 DOM → 高频渲染压力

### V4 核心改进
1. **动态 transition 管理**: 拖拽/滚轮时 `transition: none`，按钮/双击/重置时启用 (参考 obsidian-mermaid-fullscreen)
2. **requestAnimationFrame 节流**: wheel 和 mousemove 合并到下一帧写入
3. **正确的事件生命周期**: mousedown→注册 document 监听，mouseup→注销 (不再丢失)
4. **指数缩放**: wheel 使用 `zoom × 1.08` 乘法因子 (参考 anvaka/panzoom)
5. **fitScale 计算**: 双重 rAF + SVG viewBox 优先 + getBoundingClientRect 回退
6. **cleanupFns 清理模式**: 所有监听器注册时推入清理函数数组，close 时一次性移除
7. CSS: 移除 `.markvault-mermaid-canvas` 的静态 transition，改为 JS 动态管理；新增 `touch-action: none`
8. 旧的 `_panzoomSetFit()` 私有方法已删除，功能内联为闭包内的 `panzoomSetFit()`

## V4.1 缩小按钮修复 (14:44)
- **问题**: zoomOutBtn 不生效 — minZoom = fitScale 导致无法缩到比 fit 更小
- **修复**: minZoom 改为 `Math.min(0.15, fitScale)`，允许缩到最低 15%
