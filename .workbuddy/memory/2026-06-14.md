### 编辑模式 region 锚点渲染修复 v2（彻底修复）✅
- 前版修复（escapeAnchorField + 宽松正则兜底）未能解决问题
- 根因再分析：
  1. `REGION_ANCHOR_REGEX` 使用 `([^%]*)` 匹配 note 字段，note 中含 `%` 时正则完全无法匹配
  2. 宽松兜底正则也使用 `[^%]*`，同样失败
  3. 第一个宽松正则产生部分匹配后加入 `alreadyHidden`，阻止第二个宽松正则的完整匹配
  4. 阅读模式 `applyRegionStyleToSectionPrecise` 用 `buildRegionAnchor`（生成转义后锚点）搜索源文本，与原始未转义锚点不匹配 → 精确匹配失败 → 回退到 `applyRegionStyleToSection` → 给 `<ul>` 等容器块加背景 → 整段列表染色
- 修复（3 个文件）：
  1. **`region-annotation.ts`**：`REGION_ANCHOR_REGEX` 的 note 字段从 `([^%]*)` 改为 `([^\n]*)`，UUID/type/color 从 `([^:%]+)` 改为 `([^:\n]+)`。锚点不跨行，`[^\n]*` 安全匹配到 `%%` 终止符，不受 note 中 `%` 影响
  2. **`highlight-applier.ts`**：移除有缺陷的双宽松正则兜底，替换为基于子串扫描的兜底：找 `markvault-region` 行，向前找 `%` 开头、向后找 `%%` 结尾，补充隐藏未匹配范围
  3. **`main.ts`**：多处修复：
     - `highlightRegionNodes`：`isWithinSameBlock` 时跳过所有块级元素（不仅是 startBlockAncestor），只包裹文本节点
     - `applyRegionStyleToSectionPrecise`：使用 `REGION_ANCHOR_REGEX` 搜索锚点位置（而非 `buildRegionAnchor`），能正确匹配含 `%` 的旧版锚点
     - `applyRegionStyleFromStartAnchor`/`applyRegionStyleToEndAnchor`：同样改用 `REGION_ANCHOR_REGEX`
     - `applyRegionStyleToSection`：识别容器块（含块级子元素如 `<ul>`），只加点击事件不加背景类
     - `applyRegionStyleToMiddleSection`：递归处理容器块子元素，叶子块才加背景
     - `applyRegionStyleToNodes`：同上容器块逻辑
     - `hideLeakedAnchorText`：使用 `[^\n]*?` 替代 `[^%]*`，增加 `mv:i` 模式
- Build 通过
