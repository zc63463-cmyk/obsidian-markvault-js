## BugFix: 跨文件关联 ENOENT 错误 (21:49)
- **根因**: 跨文件 addRelation 触发两次 _markDirty → 两个 2s debounce timer 分别 flushFile → 并发调用 _writeIndexFile → 竞态写入/删除同一 tmpPath → 第二次 adapter.remove(tmpPath) 时文件已不存在 → ENOENT
- **修复 3 层防御**:
  1. `_writeIndexFile` / `_writeMetaFile` 新增互斥锁 (_indexWriting / _metaWriting)，并发调用等待前一个完成后再写一次
  2. 所有 `adapter.remove(tmpPath)` 改为防御性 try/catch（ENOENT 不再抛出）
  3. `flushFile` 中 `_indexDirty` 检查后先原子取走标记再写入，防止并发 flush 重复写
- 涉及文件: annotation-store.ts (_writeIndexFile, _writeMetaFile, _atomicWrite, _writeFileShard, flushFile, flushAll)
- 379/379 测试全绿，已部署
